🖥 VS Code

Éditeur de code gratuit Microsoft. Supporte toutes les extensions, terminal intégré, Git intégré. C'est le QG de tout le développement.

🤖 Claude Code

CLI d'Anthropic à installer dans le terminal. Il lit votre codebase, modifie les fichiers, commit et push — comme un développeur senior dans votre équipe.

🐙 GitHub

Hébergement du code source. Chaque git push déclenche automatiquement un déploiement Vercel.

▲ Vercel

Hébergement frontend gratuit. Déploiement automatique en ~30 secondes après chaque push. HTTPS offert, CDN mondial.

💡 Coût de la stack de base : 0 € — VS Code gratuit, GitHub gratuit, Vercel gratuit (plan Hobby), Claude Code ~20 $/mois.
📋

Prérequis & comptes à créer

1
Compte GitHub

github.com — Gratuit. Choisir un nom d'utilisateur professionnel (il apparaîtra dans les URLs de vos repos).

2
Compte Vercel

vercel.com — Se connecter directement avec son compte GitHub pour lier les deux automatiquement.

3
Compte Anthropic (Claude)

console.anthropic.com — Créer une clé API. Ajouter des crédits (minimum recommandé : 5 $). La clé commence par sk-ant-…

4
Compte Supabase (si backend)

supabase.com — Gratuit jusqu'à 2 projets. Récupérer : URL du projet + clé anon + clé service_role.

5
Git for Windows

git-scm.com/download/win — Installer avant VS Code. Cocher "Add to PATH" pendant l'installation.

6
Node.js LTS

nodejs.org — Version LTS (ex : v22+). Nécessaire pour npm et pour lancer Claude Code.

Étape 1
🖥

Installer & configurer VS Code

1
Téléchargement

code.visualstudio.com → Télécharger pour Windows. Installer avec les options par défaut (cocher "Ajouter à PATH").

2
Extensions indispensables

Ouvrir le panneau Extensions (Ctrl+Shift+X) et installer :

ESLint Prettier GitLens Tailwind CSS IntelliSense TypeScript Importer
3
Terminal intégré

Vérifier que Git Bash est disponible : Ctrl+` pour ouvrir le terminal → sélectionner "Git Bash" dans le menu déroulant.

⚙️ Paramètres recommandés

Ouvrir Ctrl+Shift+P → "Open User Settings JSON" :

// settings.json { "editor.formatOnSave": true, "editor.defaultFormatter": "esbenp.prettier-vscode", "terminal.integrated.defaultProfile.windows": "Git Bash", "git.autofetch": true }
✅ Vérification : dans le terminal VS Code, taper git --version et node --version — les deux doivent répondre.
Étape 2
🐙

Configurer Git & GitHub

1
Configurer son identité Git

Dans le terminal VS Code, une seule fois pour toute la machine :

git config --global user.name "Prénom NOM" git config --global user.email "votre@email.com"
2
Créer le dépôt sur GitHub

github.com → New repository → Nom du projet → Private ou Public → Créer sans README (on poussera depuis VS Code).

3
Authentification GitHub dans VS Code

VS Code détecte automatiquement GitHub. Premier push → une fenêtre navigateur s'ouvre pour autoriser. Accepter et c'est fait.

📁 Initialiser un projet existant

# Depuis le dossier du projet git init git add . git commit -m "Initial commit" git branch -M main git remote add origin \ https://github.com/user/repo.git git push -u origin main

🔑 Fichier .gitignore essentiel

# Jamais committer ces fichiers ! .env .env.local .env.*.local node_modules/ dist/
⚠️ Ne jamais committer de clés API dans le code. Utiliser des variables d'environnement (.env.local).
Étape 3
🤖

Installer Claude Code (CLI)

1
Installation globale via npm
npm install -g @anthropic-ai/claude-code
2
Premier lancement

Dans le terminal, depuis votre dossier projet :

claude
3
Authentification

Claude Code ouvre le navigateur vers console.anthropic.com. S'identifier → copier la clé API → coller dans le terminal. La clé est sauvegardée localement.

4
Vérification
claude --version # → @anthropic-ai/claude-code 1.x.x

🎯 Ce que Claude Code peut faire

  • Lire et modifier tous les fichiers du projet
  • Exécuter des commandes terminal
  • Créer des branches, committer, pousser
  • Chercher dans le code (grep, glob)
  • Lire la doc web pour vous aider
  • Mémoriser des préférences entre sessions
💡 Extension VS Code disponible aussi : chercher "Claude" dans les extensions. Mais le terminal reste la version la plus puissante.
ℹ️ Tarif : ~20 $/mois (plan Pro) pour une utilisation intensive de dev. Ou paiement à l'usage via la clé API.
Étape 3 (suite)
⚙️

Configurer Claude Code

📄 CLAUDE.md — La "carte de visite" du projet

À la racine du projet, ce fichier est automatiquement lu par Claude Code à chaque session. Il contient :

# CLAUDE.md ## Stack technique React + TypeScript + Vite Supabase (Edge Functions Deno) Déployé sur Vercel ## Règles de développement - Committer et pusher après chaque feature - Pas de commentaires inutiles dans le code - Variables d'env dans .env.local
💡 Plus le CLAUDE.md est précis, moins vous aurez besoin de ré-expliquer le contexte à chaque session.

🔐 Permissions (.claude/settings.json)

Gérer ce que Claude Code peut faire sans demander confirmation :

{ "permissions": { "allow": [ "Bash(git *)", "Bash(npm *)", "Bash(npx *)" ] } }

🧠 Mémoire persistante

Claude Code mémorise vos préférences entre sessions. Exemple : "retiens que je veux toujours committer et pusher après chaque développement". Il le fera automatiquement à chaque session.

Étape 4
⚛️

Créer le projet React + TypeScript

🚀 Scaffolding avec Vite

# Créer le projet npm create vite@latest mon-projet \ -- --template react-ts # Entrer dans le dossier cd mon-projet # Installer les dépendances npm install # Lancer en développement npm run dev
🔗 L'app tourne sur http://localhost:5173 avec rechargement automatique à chaque sauvegarde.

📦 Bibliothèques utilisées dans Trust Build-IA

# UI Components npm install shadcn/ui # Routing npm install react-router-dom # Supabase client npm install @supabase/supabase-js # Icônes npm install lucide-react # Notifications npm install sonner # Formulaires npm install react-hook-form
💡 Laisser Claude Code installer les dépendances pour vous : "installe shadcn/ui et configure-le". Il gère le setup complet.
Étape 5
🗄

Configurer Supabase (backend)

1
Créer le projet Supabase

supabase.com → New Project → choisir région EU West. Noter le mot de passe DB (non récupérable après).

2
Récupérer les clés

Settings → API → copier Project URL + anon public key + service_role key.

3
Variables d'environnement locales
# .env.local VITE_SUPABASE_URL=https://xxx.supabase.co VITE_SUPABASE_ANON_KEY=eyJhbGc...

🖥 Supabase CLI (Windows via Scoop)

# Installer Scoop (gestionnaire paquets) Set-ExecutionPolicy -Scope CurrentUser \ -ExecutionPolicy RemoteSigned irm get.scoop.sh | iex # Installer Supabase CLI scoop install supabase # Si commande introuvable dans VS Code $env:PATH += ";$env:USERPROFILE\scoop\shims" # Déployer une Edge Function supabase functions deploy ma-fonction \ --no-verify-jwt
⚠️ Toujours déployer les Edge Functions avec --no-verify-jwt, sinon Supabase rejette les requêtes même authentifiées.
Étape 6
▲

Déployer sur Vercel

1
Importer le repo GitHub

vercel.com → Add New Project → Import Git Repository → sélectionner le repo GitHub → Deploy.

2
Configuration du build

Vercel détecte Vite automatiquement. Framework Preset : Vite. Build Command : npm run build. Output Directory : dist.

3
Variables d'environnement

Settings → Environment Variables → ajouter les mêmes clés que dans .env.local (VITE_SUPABASE_URL, etc.).

4
SPA Routing (vercel.json)

Pour que React Router fonctionne, ajouter à la racine :

// vercel.json { "rewrites": [ { "source": "/(.*)", "destination": "/" } ] }

Services tiers — Configuration avancée

📧 SendGrid — Emails transactionnels

Alternative à Resend, sans besoin de domaine custom (Single Sender Verification sur Gmail suffit).

// Dans une Edge Function Supabase await fetch( "https://api.sendgrid.com/v3/mail/send", { method: "POST", headers: { Authorization: `Bearer ${SENDGRID_KEY}` }, body: JSON.stringify({ personalizations: [{ to: [{email}] }], from: { email: fromEmail, name }, subject: "Objet", content: [{ type: "text/html", value: html }] }) } )

⏰ pg_cron — Tâches planifiées

Pour des rappels automatiques (ex: expiration KBIS).

1. Activer l'extension : Supabase → Database → Extensions → pg_cron

-- 2. Planifier (SQL Editor) SELECT cron.schedule( 'mon-rappel', '0 8 * * 1', -- tous les lundis 8h $$SELECT net.http_post( url := 'https://xxx.supabase.co/functions/v1/ma-fn', headers := '{"Authorization":"Bearer KEY"}'::jsonb, body := '{}'::jsonb )$$ ); -- Réponse "schedule 1" = succès ✅
⚠️ Activer pg_cron avant d'exécuter le SQL — sinon erreur schema "cron" does not exist.

Workflow quotidien avec Claude Code

1
Ouvrir VS Code + lancer Claude Code

Ouvrir le terminal intégré (Ctrl+`) → taper claude. Claude lit automatiquement le projet et le CLAUDE.md.

2
Décrire la feature en langage naturel

Exemple : "Ajoute une page de paramètres avec un champ pour le préfixe des factures, sauvegarde dans la table artisan_settings"

3
Valider les modifications proposées

Claude Code montre chaque modification avant de l'appliquer. Approuver, modifier ou refuser. Il s'adapte à vos corrections.

4
Commit + Push automatique

Claude Code crée le commit avec un message descriptif et pousse sur GitHub. Vercel déploie en ~30 secondes.

5
Tester sur l'URL Vercel

L'app est en ligne. Tester sur smartphone et desktop. Si bug → revenir en étape 2 avec une description du problème.

⚡ Ce cycle complet — idée → code → prod — prend 5 à 15 minutes par feature avec cette stack.

Pièges fréquents & conseils

⚠️ Erreurs classiques

!
Clés API dans le code

Toujours utiliser .env.local. Ne jamais hardcoder une clé dans un fichier JS/TS. GitHub scanne les dépôts et révoque les clés exposées.

!
Variables VITE_* pour données secrètes

Les variables VITE_* sont visibles dans le navigateur (bundle JS). Réserver aux clés publiques Supabase uniquement.

!
Edge Functions sans --no-verify-jwt

Supabase rejette les requêtes si on oublie ce flag au déploiement. Toujours : supabase functions deploy fn --no-verify-jwt

✅ Bonnes pratiques

✓
CLAUDE.md bien documenté

Plus il contient d'infos sur l'architecture, les conventions et les règles, plus Claude Code sera autonome.

✓
Migrations SQL versionnées

Stocker toutes les modifications de schéma dans supabase/migrations/. Committer ces fichiers. Rejouer facilement sur un nouveau projet.

✓
Activer pg_cron AVANT le SQL

Supabase → Database → Extensions → pg_cron → Enable. Puis exécuter le cron.schedule().
